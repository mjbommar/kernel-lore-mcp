# Research — gix vs git2-rs (April 14 2026)

## Decision

**gix 0.81.0** (gitoxide). `ThreadSafeRepository` + rayon-per-shard
+ `with_hidden` for incremental walks. `mail-parser` 0.11 for
RFC822.

## Key facts

- **gix 0.81.** Still pre-1.0 but read paths (pack, commit-graph,
  object DB) are production-ready and match what cargo's
  gitoxide-based shell-out-replacements use in production.
- **git2-rs 0.20.4** (libgit2 1.8). Maintained but slower on
  linear-history walks for big mailing-list repos (Byron/gitoxide
  perf benchmarks, 2025). Critical deal-breaker:
  `git2::Repository` is **NOT `Sync`** — you'd need one per rayon
  worker.
- **gix `ThreadSafeRepository::to_thread_local()`** is the canonical
  pattern for rayon fanout without per-thread re-opens.

## Walking pattern

```rust
let ts = ThreadSafeRepository::open(shard_path)?;
let repo = ts.to_thread_local();
let walk = repo.rev_walk([head.id])
    .with_hidden(last_indexed_oid.into_iter())  // like `git log new ^old`
    .sorting(Sorting::ByCommitTimeNewestFirst)
    .all()?;
for info in walk { ... }
```

Incremental: `with_hidden(last_indexed_oid)` gives exactly the
new commits since our state file.

## Parallelism pattern

- **One rayon task per shard.** 390 shards >> CPU count.
- **Not within a shard.** Pack random-access thrash.
- `rayon::prelude::*; shards.par_iter().try_for_each(...)`.

## Pre-ingestion speedup

Run once per shard before first ingest (5–10× faster walks):
```
git multi-pack-index write
git commit-graph write
```
Add to grokmirror post-pull hook for new shards.

## Memory

gix mmaps packs with a tunable LRU pack cache
(`gix::open::Options::object_cache_size_if_unset`). Tune ~4 MB
when streaming 390 shards concurrently.

## Public-inbox v2 quick recap

- Commit per message. Tree has `m` (RFC822 blob) and optionally
  `d` (deletion marker).
- Commit author date ≈ message received-time.

## No existing public-inbox reader crate

Confirmed as of April 2026 (crates.io search: `public-inbox`,
`lore`, `mbox-git`). We're the first Rust project doing this.
`mail-parser` 0.11 for RFC822 decoding.
