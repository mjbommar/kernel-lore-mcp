## Decision

**tantivy 0.26.0** (crate), embed via our own PyO3 bindings (not
tantivy-py). Stemmer feature NOT enabled.

## Key facts (from tantivy CHANGELOG)

- **0.26 (unreleased as of April 14 2026, master branch).** Shipped
  items on master that materially affect us:
  - **Stemming moved behind `stemmer` feature flag (PR #2791).**
    Perfect â€” matches our no-stemming proscription. Leave the
    feature off.
  - Regex syntax in the query grammar (PRs #2677, #2818) â€” the
    built-in parser now supports `/pattern/` natively. Still slower
    than our trigram tier for code, but useful for narrow regex in
    prose.
  - `Term` split into `Term` and `IndexingTerm` (PR #2744).
  - Various perf wins: lazy scorers, `DocSet::cost()`,
    `seek_danger`.
- **0.25 (shipped).** Rust edition 2024, query parser tolerances.
- Minimum Rust 1.81 for 0.24; 0.26 continues on edition 2024.

## Why embed tantivy directly (not tantivy-py)

- tantivy-py doesn't let us register custom Rust tokenizers from
  Python. Our `kernel_prose` analyzer must live in Rust.
- We want to co-locate the Arrow/Parquet metadata tier and the
  trigram tier in the same `_native` extension module.
- Fine-grained control of `IndexWriter` memory budget and merge
  policy.

## Custom tokenizer idiom (confirmed April 2026)

`TextAnalyzer::builder(tokenizer).filter(...).build()` still the
idiom. Register via `index.tokenizers().register(name, analyzer)`.
**Tokenizer names are not persisted in `meta.json`** â€” every
process that opens the index must re-register. See
[`../indexing/tokenizer-spec.md`](../indexing/tokenizer-spec.md).

## Positions off

`IndexRecordOption::WithFreqs` (drops positions). Expected 30â€“50%
smaller postings for prose-heavy fields. Kills phrase/slop queries;
we accept that â€” see trade-offs doc.

## Alternatives we looked at and rejected

| Library | Why no |
|---|---|
| tantivy-py | Too thin for our custom-analyzer needs |
| Sonic | Daemon, schemaless, no structured fields |
| Toshi | HTTP wrapper over tantivy; adds nothing for a library user |
| Meilisearch | Typo-tolerant consumer search; wrong for code |
| Quickwit | Log engine; warns against user-facing search |
| Bluge / Bleve | Go, not Rust |
| bm25 crate | In-memory scorer, not a disk index |

## Trigram tier: build our own

No mature Rust Zoekt-like library on crates.io as of April 2026.
Build on `fst` (term dict) + `roaring` (posting lists). ~400 LOC
of index-writer + reader. See
[`../indexing/trigram-tier.md`](../indexing/trigram-tier.md).
